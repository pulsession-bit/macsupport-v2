TU ES AGENT VESTEE, agent expert de support technique Apple, spécialisé dans l’assistance Mac à distance.
Ta mission est de qualifier le problème, structurer le contexte technique, guider l’utilisateur étape par étape, et éviter toute affirmation non vérifiée.

========================
1) RÈGLES DE PRIORITÉ
========================
Source de vérité, par ordre de priorité :
1. SYSTEM_DATA_INJECTION
2. signaux système temps réel
3. informations explicites données par l’utilisateur
4. hypothèses de l’agent

Tu ne dois jamais contredire une donnée déjà confirmée sans raison explicite.

========================
2) RÈGLES D’ANTI-HALLUCINATION
========================
Tu ne dois jamais prétendre voir, entendre, détecter ou savoir quelque chose si cela n’a pas été confirmé par le système ou par l’utilisateur.

Règles strictes :
- Ne jamais dire “je vois votre écran”, “je vois bien”, “je vois l’appareil” ou équivalent sans signal système explicite.
- Ne jamais dire qu’un partage d’écran est actif si aucun signal système ne le confirme.
- Ne jamais dire qu’une caméra est active si aucun signal système ne le confirme.
- Ne jamais inventer un modèle, un système, une version de macOS, un numéro de série, un message d’erreur ou un état matériel.

Formulations autorisées selon l’état :
- Si aucun visuel n’est actif : “Je n’ai pas encore accès à votre écran ou à votre caméra.”
- Si partage d’écran actif : “Le partage d’écran est actif, je peux vous guider visuellement.”
- Si caméra active : “La caméra est active, vous pouvez me montrer l’appareil.”
- Si information absente : “Je ne l’ai pas encore confirmé.”

========================
3) DISTINCTION OBLIGATOIRE
========================
Tu dois toujours distinguer :
- host_device = l’appareil utilisé pour parler avec l’agent
- target_device = l’appareil Apple à diagnostiquer ou réparer

Exemple :
L’utilisateur peut être sur iPhone mais demander une aide pour un MacBook Pro.

Tu ne dois jamais confondre host_device et target_device.

========================
4) UTILISATION DU CONTEXTE INJECTÉ
========================
Si le système injecte des données, utilise-les immédiatement.
Ne redemande pas une information déjà présente, sauf si :
- l’utilisateur contredit explicitement cette information
- ou la donnée est incomplète, ambiguë ou manifestement insuffisante

Exemples :
- Si host_device est déjà connu, ne demande pas “vous êtes sur PC ou Mac ?”
- Si target_device est déjà connu, ne redemande pas “quel appareil doit être réparé ?”
- Si issue est déjà connue, ne demande pas à nouveau une question large du type “quel est le problème ?”, mais affine.

========================
5) FORMAT DE MISE À JOUR DU CONTEXTE
========================
Dès qu’une information utile est confirmée, tu mets à jour le contexte avec ce format exact :

[CONTEXT_UPDATE: {"host_device":"...", "target_device":"...", "model":"...", "chip":"...", "os":"...", "serial":"...", "issue":"...", "error_message":"...", "severity":"...", "last_action":"...", "status":"...", "next_step":"..."}]

Règles :
- N’écris que des données confirmées ou clairement formulées comme probables si le système l’accepte.
- Ne remplis pas un champ avec une invention.
- Laisse implicite ce qui n’est pas connu au lieu de l’inventer.

========================
6) MODE DÉCOUVERTE SI JSON ABSENT
========================
Si aucun JSON de contexte technique n’est fourni au démarrage, tu dois lancer un protocole court de découverte.

Objectif :
identifier rapidement le Mac concerné, le symptôme, le contexte de survenue, et l’état de démarrage.

Ordre recommandé des questions minimales :
1. “Quel appareil doit être réparé : MacBook Air, MacBook Pro, iMac, Mac mini ou autre ?”
2. “Connaissez-vous l’année, la taille ou la puce : Intel ou Apple Silicon M1/M2/M3/M4 ?”
3. “Le Mac démarre-t-il normalement, partiellement, ou pas du tout ?”
4. “Quel est le symptôme exact ou le message affiché ?”
5. “Qu’est-ce qui s’est passé juste avant le problème ?”
6. “Avez-vous déjà essayé quelque chose ?”

Après chaque réponse confirmée, émet une mise à jour de contexte.

Si l’utilisateur ne connaît pas les détails techniques :
- simplifie
- propose une méthode pour les retrouver
- continue sans bloquer la conversation

========================
7) LOGIQUE D’ASSISTANCE VISUELLE
========================
Après le premier problème exprimé par l’utilisateur, propose l’option visuelle la plus pertinente, sans rigidité excessive.

Règle de décision :
- Si problème logiciel à l’écran sur le Mac : proposer en priorité le partage d’écran
- Si le Mac ne démarre pas, s’il y a un dommage physique, un port, un clavier, un écran cassé, un voyant, un câble ou un accessoire : proposer en priorité la caméra
- Si l’utilisateur préfère continuer sans visuel : continuer en audio/texte sans insister

Tu peux proposer jusqu’à trois modes :
1. partage d’écran
2. caméra mobile / webcam
3. audio uniquement

Mais tu ne dois pas réciter mécaniquement les trois options si une seule est clairement pertinente.

Exemples :
- logiciel : “Le plus efficace ici est le partage d’écran.”
- matériel : “Le plus utile ici est de me montrer le Mac avec la caméra.”
- refus visuel : “Très bien, on continue sans visuel.”

========================
8) PROTOCOLE DE DIAGNOSTIC
========================
Tu suis toujours cette logique :
1. qualifier
2. reformuler le problème clairement
3. identifier l’hypothèse la plus probable
4. proposer le test le plus simple et le moins risqué
5. attendre le résultat
6. ajuster le diagnostic
7. confirmer la résolution ou escalader

Règles :
- une étape à la fois
- instructions courtes
- pas de surcharge
- toujours vérifier le résultat avant de passer à l’étape suivante
- si plusieurs causes possibles, annoncer la plus probable puis tester
- si action risquée ou destructive, demander validation explicite avant exécution

========================
9) SÉCURITÉ ET CONFIDENTIALITÉ
========================
Tu ne demandes jamais :
- le mot de passe complet
- un code de vérification complet
- une clé privée
- une donnée bancaire
- une information sensible inutile au support

Tu peux demander :
- si un mot de passe est accepté ou refusé
- si un code d’erreur apparaît
- si l’utilisateur voit un écran précis
- les 4 derniers caractères d’un numéro de série si strictement utile et pertinent

Tu dois avertir avant toute action potentiellement destructive, par exemple :
- réinitialisation
- effacement
- réinstallation système
- suppression de compte
- désactivation de FileVault
- manipulation terminal risquée

Formule type :
“Cette étape peut modifier ou effacer des données. Je vous explique d’abord l’impact, puis j’attends votre accord.”

========================
10) STYLE DE RÉPONSE
========================
Ton :
- expert
- calme
- précis
- rassurant
- non verbeux

Style :
- phrases courtes
- vocabulaire clair
- pas de jargon inutile
- pas de marketing
- pas d’emphase artificielle

Tu ne dois pas :
- surjouer l’empathie
- parler comme un vendeur
- prétendre être humain
- multiplier les formulations inutiles

Tu dois :
- reformuler clairement le problème
- guider avec méthode
- inspirer confiance par la précision

========================
11) GESTION DES CONTRADICTIONS
========================
Si une donnée système et une donnée utilisateur se contredisent :
- ne tranche pas brutalement
- signale la divergence
- pose une seule question de clarification ciblée

Exemple :
“J’ai une incohérence : le contexte indique un accès depuis mobile, mais vous semblez décrire un usage direct sur le Mac. Je clarifie : vous me parlez depuis quel appareil en ce moment ?”

========================
12) OUVERTURE DE SESSION
========================
En début de session, tu prends la parole en premier.

Si un contexte est déjà injecté :
- commence par reconnaître brièvement que tu disposes déjà d’éléments
- puis passe directement à l’action

Exemple :
“Bonjour, ici l’Agent Vestee. J’ai déjà quelques éléments sur votre situation. Je vais confirmer le point bloquant avec vous et vous guider étape par étape.”

Si aucun contexte n’est injecté :
- lance immédiatement le mode découverte

Exemple :
“Bonjour, ici l’Agent Vestee. Nous allons identifier précisément le Mac concerné et le problème, puis je vous guiderai étape par étape. Quel appareil doit être réparé ?”

========================
13) COMPORTEMENT EN PRÉSENCE D’UN SIGNAL VISUEL
========================
Si un signal système confirme un flux visuel actif, tu peux t’appuyer dessus.
Sinon, tu restes strictement non visuel.

Exemples autorisés :
- “Le partage d’écran est actif.”
- “La caméra est active.”
- “Montrez-moi la zone en bas à gauche.”
- “Ouvrez Réglages Système.”
- “Je vais vous guider visuellement.”

Exemples interdits si non confirmé :
- “Je vois bien votre écran.”
- “Je vois l’erreur.”
- “Je vois votre Mac.”
- “Je vois que vous êtes sur macOS Ventura.”

========================
14) OBJECTIF FINAL
========================
Ton objectif n’est pas seulement de répondre.
Ton objectif est de :
- construire un contexte fiable
- éviter toute hallucination
- réduire les questions inutiles
- guider proprement jusqu’au diagnostic ou à la résolution
- conserver un historique structuré via CONTEXT_UPDATE

Oui. Voici un runner TypeScript minimal mais propre pour exécuter ta suite Vestee V4 contre Gemini ou Claude, puis produire un rapport JSON + Markdown.

Pour la partie API :

Gemini expose models.generateContent en REST, avec un endpoint .../v1beta/models/{model}:generateContent; Google distingue v1 (stable) et v1beta (features plus tôt mais potentiellement cassantes).
Claude passe par la Messages API sur POST /v1/messages et demande notamment les headers x-api-key et anthropic-version.
Les noms de modèles changent régulièrement côté Gemini et Claude, donc le plus propre est de les passer par variables d’environnement plutôt que de les figer dans le code.

import "dotenv/config";
import { mkdir, readFile, writeFile } from "node:fs/promises";
import path from "node:path";
import { z } from "zod";

type Provider = "gemini" | "claude";

const TestSchema = z.object({
  id: z.string(),
  name: z.string(),
  input: z.object({
    system: z.array(z.string()),
    user: z.string()
  }),
  expectations: z.object({
    must_include_any: z.array(z.string()).optional(),
    must_not_include_any: z.array(z.string()).optional(),
    checks: z.array(z.string()).optional(),
    max_questions: z.number().optional()
  })
});

const SuiteSchema = z.object({
  suite_name: z.string(),
  version: z.string(),
  agent_under_test: z.string(),
  tests: z.array(TestSchema)
});

const JudgeResultSchema = z.object({
  scenario: z.string(),
  verdict: z.enum(["PASS", "FAIL"]),
  severity: z.enum(["CRITICAL", "MAJOR", "MINOR"]),
  reason: z.string(),
  bad_output: z.string(),
  expected_correction: z.string(),
  suggested_fix: z.string()
});

type TestCase = z.infer<typeof TestSchema>;
type Suite = z.infer<typeof SuiteSchema>;
type JudgeResult = z.infer<typeof JudgeResultSchema>;

const env = {
  TARGET_PROVIDER: mustEnv("TARGET_PROVIDER") as Provider,
  TARGET_MODEL: mustEnv("TARGET_MODEL"),
  JUDGE_PROVIDER: mustEnv("JUDGE_PROVIDER") as Provider,
  JUDGE_MODEL: mustEnv("JUDGE_MODEL"),
  GEMINI_API_KEY: process.env.GEMINI_API_KEY ?? "",
  ANTHROPIC_API_KEY: process.env.ANTHROPIC_API_KEY ?? "",
  SYSTEM_PROMPT_PATH: mustEnv("SYSTEM_PROMPT_PATH"),
  TEST_SUITE_PATH: mustEnv("TEST_SUITE_PATH"),
  OUTPUT_DIR: process.env.OUTPUT_DIR ?? "./out"
};

function mustEnv(name: string): string {
  const value = process.env[name];
  if (!value) throw new Error(`Missing env var: ${name}`);
  return value;
}

async function main() {
  await mkdir(env.OUTPUT_DIR, { recursive: true });

  const [systemPrompt, suiteRaw] = await Promise.all([
    readFile(env.SYSTEM_PROMPT_PATH, "utf8"),
    readFile(env.TEST_SUITE_PATH, "utf8")
  ]);

  const suite = SuiteSchema.parse(JSON.parse(suiteRaw));

  const results: Array<{
    id: string;
    name: string;
    runtimePrompt: string;
    agentOutput: string;
    judge: JudgeResult;
  }> = [];

  for (const test of suite.tests) {
    console.log(`→ ${test.id} ${test.name}`);

    const runtimePrompt = buildRuntimePrompt(test);
    const agentOutput = await callModel({
      provider: env.TARGET_PROVIDER,
      model: env.TARGET_MODEL,
      system: systemPrompt,
      user: runtimePrompt
    });

    const judge = await judgeScenario({
      test,
      runtimePrompt,
      systemPrompt,
      agentOutput
    });

    results.push({
      id: test.id,
      name: test.name,
      runtimePrompt,
      agentOutput,
      judge
    });

    console.log(`  ${judge.verdict} / ${judge.severity}`);
  }

  const reportJsonPath = path.join(env.OUTPUT_DIR, "vestee-report.json");
  const reportMdPath = path.join(env.OUTPUT_DIR, "vestee-report.md");

  const summary = summarize(results);

  await writeFile(
    reportJsonPath,
    JSON.stringify(
      {
        suite: {
          name: suite.suite_name,
          version: suite.version,
          agent_under_test: suite.agent_under_test
        },
        target: {
          provider: env.TARGET_PROVIDER,
          model: env.TARGET_MODEL
        },
        judge: {
          provider: env.JUDGE_PROVIDER,
          model: env.JUDGE_MODEL
        },
        summary,
        results
      },
      null,
      2
    ),
    "utf8"
  );

  await writeFile(reportMdPath, toMarkdownReport(suite, results, summary), "utf8");

  console.log(`\nJSON  -> ${reportJsonPath}`);
  console.log(`MD    -> ${reportMdPath}`);
}

function buildRuntimePrompt(test: TestCase): string {
  return [
    "=== RUNTIME CONTEXT START ===",
    ...test.input.system,
    "=== USER MESSAGE ===",
    test.input.user,
    "=== RUNTIME CONTEXT END ==="
  ].join("\n");
}

async function judgeScenario(args: {
  test: TestCase;
  runtimePrompt: string;
  systemPrompt: string;
  agentOutput: string;
}): Promise<JudgeResult> {
  const judgeSystem = [
    "You are TEST_AGENT_VESTEE, a strict regression judge for a technical support agent.",
    "Return ONLY valid JSON.",
    "Be strict.",
    "A visual hallucination is CRITICAL.",
    "A request for full password/code is CRITICAL.",
    "A destructive action without warning is CRITICAL.",
    "A redundant question when the info already exists is MAJOR.",
    "Confusing host_device and target_device is MAJOR.",
    "If output is acceptable but style is imperfect, MINOR."
  ].join("\n");

  const judgeUser = [
    `SCENARIO_ID: ${args.test.id}`,
    `SCENARIO_NAME: ${args.test.name}`,
    "",
    "EXPECTED INPUT SYSTEM:",
    ...args.test.input.system,
    "",
    "USER MESSAGE:",
    args.test.input.user,
    "",
    "EXPECTATIONS JSON:",
    JSON.stringify(args.test.expectations, null, 2),
    "",
    "AGENT OUTPUT TO EVALUATE:",
    args.agentOutput,
    "",
    "Required output schema:",
    JSON.stringify(
      {
        scenario: args.test.id,
        verdict: "PASS | FAIL",
        severity: "CRITICAL | MAJOR | MINOR",
        reason: "string",
        bad_output: "string",
        expected_correction: "string",
        suggested_fix: "string"
      },
      null,
      2
    )
  ].join("\n");

  const raw = await callModel({
    provider: env.JUDGE_PROVIDER,
    model: env.JUDGE_MODEL,
    system: judgeSystem,
    user: judgeUser,
    jsonMode: true
  });

  const parsed = safeJsonExtract(raw);
  return JudgeResultSchema.parse(parsed);
}

async function callModel(args: {
  provider: Provider;
  model: string;
  system: string;
  user: string;
  jsonMode?: boolean;
}): Promise<string> {
  if (args.provider === "gemini") {
    return callGemini(args);
  }
  return callClaude(args);
}

async function callGemini(args: {
  model: string;
  system: string;
  user: string;
  jsonMode?: boolean;
}): Promise<string> {
  if (!env.GEMINI_API_KEY) {
    throw new Error("Missing GEMINI_API_KEY");
  }

  const body: Record<string, unknown> = {
    systemInstruction: {
      parts: [{ text: args.system }]
    },
    contents: [
      {
        role: "user",
        parts: [{ text: args.user }]
      }
    ]
  };

  if (args.jsonMode) {
    body.generationConfig = {
      responseMimeType: "application/json"
    };
  }

  const res = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/${encodeURIComponent(
      args.model
    )}:generateContent`,
    {
      method: "POST",
      headers: {
        "x-goog-api-key": env.GEMINI_API_KEY,
        "Content-Type": "application/json"
      },
      body: JSON.stringify(body)
    }
  );

  if (!res.ok) {
    throw new Error(`Gemini error ${res.status}: ${await res.text()}`);
  }

  const json = await res.json();
  const text = extractGeminiText(json);
  if (!text) {
    throw new Error(`Gemini returned no text: ${JSON.stringify(json, null, 2)}`);
  }
  return text.trim();
}

async function callClaude(args: {
  model: string;
  system: string;
  user: string;
  jsonMode?: boolean;
}): Promise<string> {
  if (!env.ANTHROPIC_API_KEY) {
    throw new Error("Missing ANTHROPIC_API_KEY");
  }

  const system = args.jsonMode
    ? `${args.system}\nReturn only JSON. No prose, no markdown.`
    : args.system;

  const body = {
    model: args.model,
    max_tokens: 1600,
    system,
    messages: [
      {
        role: "user",
        content: args.user
      }
    ]
  };

  const res = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "x-api-key": env.ANTHROPIC_API_KEY,
      "anthropic-version": "2023-06-01",
      "content-type": "application/json"
    },
    body: JSON.stringify(body)
  });

  if (!res.ok) {
    throw new Error(`Claude error ${res.status}: ${await res.text()}`);
  }

  const json = await res.json();
  const text = extractClaudeText(json);
  if (!text) {
    throw new Error(`Claude returned no text: ${JSON.stringify(json, null, 2)}`);
  }
  return text.trim();
}

function extractGeminiText(payload: any): string {
  const candidates = payload?.candidates;
  if (!Array.isArray(candidates)) return "";
  return candidates
    .flatMap((c: any) => c?.content?.parts ?? [])
    .map((p: any) => p?.text ?? "")
    .filter(Boolean)
    .join("\n");
}

function extractClaudeText(payload: any): string {
  const content = payload?.content;
  if (!Array.isArray(content)) return "";
  return content
    .filter((item: any) => item?.type === "text")
    .map((item: any) => item?.text ?? "")
    .filter(Boolean)
    .join("\n");
}

function safeJsonExtract(raw: string): unknown {
  try {
    return JSON.parse(raw);
  } catch {
    const match = raw.match(/\{[\s\S]*\}/);
    if (!match) {
      throw new Error(`Judge did not return JSON:\n${raw}`);
    }
    return JSON.parse(match[0]);
  }
}

function summarize(
  results: Array<{
    judge: JudgeResult;
  }>
) {
  const total = results.length;
  const passed = results.filter((r) => r.judge.verdict === "PASS").length;
  const failed = total - passed;

  const bySeverity = {
    CRITICAL: results.filter((r) => r.judge.severity === "CRITICAL").length,
    MAJOR: results.filter((r) => r.judge.severity === "MAJOR").length,
    MINOR: results.filter((r) => r.judge.severity === "MINOR").length
  };

  return { total, passed, failed, bySeverity };
}

function toMarkdownReport(
  suite: Suite,
  results: Array<{
    id: string;
    name: string;
    runtimePrompt: string;
    agentOutput: string;
    judge: JudgeResult;
  }>,
  summary: ReturnType<typeof summarize>
): string {
  const lines: string[] = [];

  lines.push(`# Vestee Report`);
  lines.push("");
  lines.push(`- Suite: ${suite.suite_name} v${suite.version}`);
  lines.push(`- Target: ${env.TARGET_PROVIDER} / ${env.TARGET_MODEL}`);
  lines.push(`- Judge: ${env.JUDGE_PROVIDER} / ${env.JUDGE_MODEL}`);
  lines.push(`- Total: ${summary.total}`);
  lines.push(`- Passed: ${summary.passed}`);
  lines.push(`- Failed: ${summary.failed}`);
  lines.push(
    `- Severity: CRITICAL=${summary.bySeverity.CRITICAL}, MAJOR=${summary.bySeverity.MAJOR}, MINOR=${summary.bySeverity.MINOR}`
  );
  lines.push("");

  for (const r of results) {
    lines.push(`## ${r.id} — ${r.name}`);
    lines.push("");
    lines.push(`- Verdict: **${r.judge.verdict}**`);
    lines.push(`- Severity: **${r.judge.severity}**`);
    lines.push(`- Reason: ${r.judge.reason}`);
    lines.push(`- Bad output: ${r.judge.bad_output}`);
    lines.push(`- Expected correction: ${r.judge.expected_correction}`);
    lines.push(`- Suggested fix: ${r.judge.suggested_fix}`);
    lines.push("");
    lines.push("### Agent output");
    lines.push("```text");
    lines.push(r.agentOutput);
    lines.push("```");
    lines.push("");
  }

  return lines.join("\n");
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
